■ ネットワーク層
・IP：Internet Protocol
・IPヘッダ：データの送受信単位であるIPデータグラム（パケット）の先頭にある、一定の形式に従って制御情報などを記載した部分
・ICMP：IP通信のエラー通知や制御メッセージを転送するプロトコル
・ICMPメッセージ：問い合わせやエラーに関するメッセージをやり取りする。IP通信の判定、フロー制御、Ping要求・応答、時間経過、Redirect機能、アドレスマスク要求・応答など。
・IPアドレス：Internet Protocol Address。インターネットで通信するための住所。通信相手を指定する際に使用
　・オクテット：IPアドレスをドットで区切った各パートを『オクテット』と呼び、左から『第1オクテット』〜『第4オクテット』と表す。1オクテット（octet）は、8ビットを表す
　・ネットワーク部：そのIPアドレスが属しているネットワークを識別するための部分
　・ホスト部：ネットワーク内のコンピュータを識別するための部分
　・クラス：ネットワークを構成するパソコンの台数に応じて、３つのクラスに分かれます。 大規模なネットワークならクラスＡ、中規模ならクラスＢ、小規模の場合はクラスＣ
　・プライベートIPアドレス：会社や家庭などの組織内（ローカル）で一意に割り当てられるIPアドレス
　・グローバルIPアドレス：インターネットに接続されているコンピュータや通信機器を個々に特定するための、一意で割り当てられたIPアドレス
・ネットワークアドレス：あるネットワークに割り当てられたIPアドレス群の中で、そのネットワーク自身を指す示すアドレス
・ブロードキャストアドレス：ネットワーク内のすべての機器（ノード）にデータを一斉配信するために使われる特殊なアドレス
・ループバックアドレス：IPアドレスの中で自らを指すもの
・サブネット：大規模なネットワーク内の小さなネットワーク
・サブネットマスク：ネットワークの範囲を定義するために使用。IPv4のIPアドレスを使用している場合のみに使用。
　・プレフィックス表記：CIDR表記。192.168.1.0/24のように、／の後にネットワークアドレスの長さを「プレフィックス」として指定します
・デフォルトゲートウェイ：家庭や社内でのLANからインターネットなどの外部ネットワークに接続する際に、通信の出入口の役割を果たす仕組み
・IPv6：「Internet Protocol Version 6」の略で、IPv4に次ぐインターネットプロトコル（ Internet Protocol ）の規格。サブネットプレフィックスとインターフェースIDから構成される。
　・サブネットプレフィックス：通常64ビット。サブネットプレフィックスの長さが、IPv4のサブネットマスクに相当
　・インターフェイスID：リンク上でホストを識別するために使用。
 

・ルータ
　・コンソール接続：機器とPCを直接ケーブルで接続し、ネットワーク機器を操作できる状態にする
　・VTY接続：ルータなどの通信機器にネットワークを通じて別のコンピュータから接続し、操作や管理を行うための仮想的な接続口
　・AUX接続：モデムを通じて公衆回線（電話回線/ISDN回線など）を利用して機器にアクセスする方法
　・ルーティングテーブル：ルータで設定されたルーティングプロトコルにより、自動的に追加されるルートのこと
　　　・ネクストホップ：ルータなどが持つ経路表（ルーティングテーブル）に記載される情報の一つで、個々の宛先ネットワーク毎に決められた、次に転送すべき隣接ルータ（のIPアドレス）のこと
　・ARPテーブル：ARP（Address Resolution Protocol）によって割り出したIPアドレスとMACアドレス（EthernetやWi-Fiなどの物理アドレス）の対応表
　・ARP：Address Resolution Protocol。IPアドレスからEthernetのMACアドレスの情報を得られる プロトコル
　・コアルータ：通信キャリアやISPネットワークの通信を中継するために利用されるルータ。
　・センタールータ：遠距離回線を挟んで中心側の装置をセンタールータと呼ぶ
　・エッジルータ：遠距離回線を挟んで中心側と末端側の両方の装置をエッジルータと呼ぶ
　・ブロードバンドルータ：家庭などで通信事業者のブロードバンド回線を通じてインターネットなどを利用する際に用いるルータ
 

・ルーティング：ネットワーク上でデータを転送する際、その経路を導き出すこと
　・直接接続：ルータと隣接しているネットワークを自動で登録する方法
　・スタティックルーティング：管理者が宛先ネットワークへの最適なルートを手動で設定したルートのこと
　・ダイナミックルーティング：自分のルーティングテーブルに隣のルータのルーティングテーブルを追加していく方式
　　・リンクステート型：すべてのルータから情報を集め，現在のネットワークの構成を知る方式
　　・ハイブリッド型：ディスタンスベクタ型とリンクステート型の両方の利点を兼ね備えたプロトコル
 

■ トランスポート層：データ伝送の信頼性を提供する役割を持つ。送信元から送り出されたデータが送信先に正しく確実に届けられる通信を提供する。
・コネクション型通信：通信に先立って、あらかじめ通信路の確保（オープン）を行い、データを送受信したあと、通信路の解放処理（クローズ）を行う形式の通信
　・ACK：アック。通信における肯定的な応答で、一般には、通信がエラーなく到達したことを示す信号
・コネクションレス型通信：相手との接続（コネクション）確立を行わずデータを送信する方式
　・TCP：コネクション型
　・スリーウェイハンドシェイク：IPネットワーク上でTCPによる接続（コネクション）を確立するための手順。 パケットの送受信を3回行うことからこのように呼ばれる。
　・Telnet：ネットワークに接続された機器を遠隔操作するために使用するアプリケーション層プロトコル。オフィスのデスクにいながら、マシンルームにあるサーバ、ルータ等の機器をパソコン上で操作ができる。
　・HTTP：WEBブラウザとの通信に用いられる
　・FTP：ネットワーク上でファイルを送受信する際に使用する通信プロトコル
　・SMTP：インターネットなどのTCP / IPネットワークで標準的に用いられる、メールを送信するための通信プロトコル
・TCPヘッダ：TCPセグメントの先頭部分にある、制御情報などを一定の形式で記述した領域
・UDP：インターネット上で標準的に利用されている通信プロトコルの一種。 リアルタイム性が重視される音声通話や動画配信などで利用される。
　・RTP：TCP/IPネットワーク上で音声や動画のように連続するデータの流れをリアルタイムに伝送するための通信プロトコル
　・SNMP：「Simple Network Management Protocol」ネットワーク機器をネットワーク経由で監視するためのプロトコル
　・DHCP：「Dynamic Host Configuration Protocol」LAN上のパソコンなどが起動すると、その都度、IPアドレスなどのネットワーク利用に必要な設定情報を、本商品から各パソコンなどに自動的に割り当てる機能
　・TFTP：「Trivial File Transfer Protocol)」特定のコンピュータ間でファイル転送する時に使用する アプリケーション層プロトコル
　・RIP：「Routing Information Protocol」ルーターなどの制御機器が経路の情報を互いに交換し合うためのプロトコル
・UDPヘッダ：UDPデータグラムの先頭部分にある、制御情報などを一定の形式で記述した領域
